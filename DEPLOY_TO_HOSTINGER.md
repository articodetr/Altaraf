# دليل رفع التطبيق على Hostinger

هذا الدليل يشرح كيفية بناء ملف APK للأندرويد ورفعه على Hostinger لتمكين المستخدمين من تحميله.

## المتطلبات

1. حساب Expo (مجاني)
2. حساب Hostinger مع خدمة استضافة
3. Node.js و npm مثبتين على جهازك

## الخطوة 1: تثبيت EAS CLI

```bash
npm install -g eas-cli
```

## الخطوة 2: تسجيل الدخول إلى Expo

```bash
eas login
```

سيطلب منك إدخال بريدك الإلكتروني وكلمة المرور لحساب Expo. إذا لم يكن لديك حساب، قم بإنشائه من:
https://expo.dev/signup

## الخطوة 3: إعداد المشروع

```bash
eas build:configure
```

هذا الأمر سيسألك عدة أسئلة. اختر:
- Platform: Android
- إذا سأل عن Google Services JSON، اختر "No"

## الخطوة 4: بناء ملف APK للإنتاج

```bash
eas build --platform android --profile production
```

**ملاحظات مهمة:**
- البناء سيتم على خوادم Expo السحابية (مجاناً)
- قد يستغرق البناء من 10-20 دقيقة
- ستحصل على رابط لتتبع حالة البناء
- عند الانتهاء، ستحصل على رابط لتحميل ملف APK

## الخطوة 5: تحميل ملف APK

بعد اكتمال البناء:

1. افتح الرابط الذي أعطاه لك EAS
2. قم بتحميل ملف `app.apk`
3. احفظ الملف على جهازك

## الخطوة 6: رفع الملفات على Hostinger

### طريقة 1: استخدام File Manager في Hostinger

1. سجل دخول إلى لوحة تحكم Hostinger (hPanel)
2. اذهب إلى **File Manager**
3. افتح مجلد `public_html` (أو المجلد الذي تريد استخدامه)
4. قم برفع الملفات التالية:
   - `download.html` - صفحة التحميل
   - `app.apk` - ملف التطبيق (الذي حملته من EAS)

### طريقة 2: استخدام FTP

1. احصل على بيانات FTP من Hostinger:
   - Host: ftp.yourdomain.com
   - Username: your_username
   - Password: your_password

2. استخدم برنامج FTP مثل FileZilla:
   - قم بالاتصال بالخادم
   - افتح مجلد `public_html`
   - ارفع الملفات:
     - `download.html`
     - `app.apk`

## الخطوة 7: الوصول إلى صفحة التحميل

بعد رفع الملفات، يمكن للمستخدمين الوصول إلى صفحة التحميل عبر:

```
https://yourdomain.com/download.html
```

أو إذا رفعت الملفات في مجلد فرعي:

```
https://yourdomain.com/app/download.html
```

## نصائح إضافية

### جعل الرابط أقصر وأسهل

يمكنك إعادة تسمية `download.html` إلى `index.html` ووضعها في مجلد فرعي مثل `app`:

```
public_html/
  └── app/
      ├── index.html
      └── app.apk
```

الآن الرابط سيكون:
```
https://yourdomain.com/app/
```

### حماية صفحة التحميل بكلمة مرور (اختياري)

إذا كنت تريد جعل التطبيق متاحاً فقط لأشخاص محددين:

1. في Hostinger File Manager
2. اذهب إلى المجلد الذي يحتوي على الملفات
3. اضغط على **Password Protect**
4. قم بإنشاء اسم مستخدم وكلمة مرور

### تحديث التطبيق

عندما تريد نشر نسخة جديدة:

1. قم بتحديث `version` في `app.json`
2. قم بتحديث `versionCode` في `app.json` (Android)
3. نفذ الأمر:
   ```bash
   eas build --platform android --profile production
   ```
4. حمل ملف APK الجديد
5. استبدل ملف `app.apk` القديم على Hostinger

## معلومات هامة

### حجم الملف

- ملف APK سيكون بحجم تقريبي 50-80 MB
- تأكد من أن استضافة Hostinger لديك تدعم رفع ملفات بهذا الحجم
- إذا كان هناك قيود، يمكنك استخدام FTP لرفع ملفات كبيرة

### أمان التطبيق

- التطبيق يستخدم HTTPS للاتصال بـ Supabase
- جميع البيانات الحساسة محمية
- يحتوي على نظام PIN لحماية الوصول

## استكشاف الأخطاء

### خطأ "مصدر غير معروف" على Android

عند محاولة تثبيت التطبيق، قد يظهر تحذير. على المستخدم:
1. فتح **الإعدادات** > **الأمان**
2. تفعيل **السماح بالتثبيت من مصادر غير معروفة**
3. أو السماح للمتصفح فقط بتثبيت التطبيقات

### لا يمكن تحميل ملف APK

تأكد من:
- الملف موجود في المسار الصحيح على الخادم
- أذونات الملف صحيحة (644 أو 755)
- رابط التحميل في `download.html` يشير إلى المسار الصحيح

### بناء APK فشل على EAS

تأكد من:
- ملف `app.json` يحتوي على معلومات صحيحة
- `package` في Android فريد (com.yourcompany.yourapp)
- جميع التبعيات في `package.json` محدثة
